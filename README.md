## 객체와 타입

### 변수
```ts
let a = 10
let s = 'my age'
let result = `${s} is ${a}`
console.log(result) // my age is 10

let result2 = `${b} is ${a}`
console.log(result2) // my age is 10
```
- `let 변수: 타입 (= 초깃값)`이 기본이지만 초깃값에 따라 타입이 정해짐
- `const`는 상수로 초깃값 필요
- `any` 타입은 다 가능
- `undefined`는 타입이자 값이다
- 하위 타입의 변수에는 상위 타입의 값을 넣을 수 없다
- 템플릿 문자열은 f스트링 느낌으로 역따옴표와 `${}`를 사용

### 인터페이스
#### 구조
```ts
interface 이름 {
    속성 이름[?]: 속성 타입[,...]
}
```
- 이 기본 틀이며 객체(변수)의 타입을 정의하는 것이 목적이다

#### 기능
- 속성에 벗어나는 객체를 막기위함 → 버그 막기 쉬움, 타입스크립트의 장점
- 속성 이름 뒤에 `?`를 붙이면 선택 속성이 되고, 있어도 없어도 된다

### 클래스
#### 구조
```ts
class 이름 {
    [private | protected | public] 속성 이름[?]: 속성 타입[...]
}
```
- `let 변수: 이름 = new 이름()` → C언어로 치면 구조체명 변수
- `변수.속성`은 C언어랑 동일
- 접근제한자는 `public`, `private`, `protected`가 있고 각각 공용, 외부에서 접근 금지, 자식까지 가능
- 생략하면 `public`으로 간주
- 생성자는 클래스의 편의를 위해 만들어졌으며 속성을 미리 선언하지 않았을 시 접근제한자가 필수

#### 인터페이스와 클래스의 차이점
- 인터페이스는 틀의 역할로 똑같은 모양을 가진 객체 변수를 만들 때 사용
- 클래스는 실제 클래스를 복사한 변수를 만들 때 사용
- `implements`를 이용해 클래스에 인터페이스 적용 가능

#### 추상 클래스
```ts
abstract class 이름 {
    abstract 속성: 타입
}
```
- 추상클래스는 상속을 위해 쓰임

#### extends
```ts
class 이름 extends 부모 클래스 이름
```
- `extends`를 통해 abstract 클래스를 상속 받을 수 있음
- `super`를 통해 부모 클래스와 상호작용 가능

#### static
- 변수를 만들어 속성에 접근할 필요 없이 `(클래스 이름).(static 변수)`로 접근 가능
